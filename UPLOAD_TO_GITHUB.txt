🌐 EASIEST METHOD: GitHub Web Interface

STEP 1: Create Repository
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. Go to: https://github.com/new

2. Fill in:
   Repository name: error-categorization-portal
   Description: AI-powered error categorization with 126 categories
   ☑ Public (so it works with GitHub Pages)
   ☐ Add a README file (uncheck - we have one)

3. Click "Create repository"

STEP 2: Upload Files
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. Click "uploading an existing file"

2. Drag and drop these 4 files:
   • index.html
   • app.js
   • categories.js
   • README.md

3. Commit message: "Initial commit: Error Categorization Portal"

4. Click "Commit changes"

STEP 3: Enable GitHub Pages
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. Go to: Settings (in your repository)

2. Click: Pages (in left sidebar)

3. Under "Source":
   Branch: main
   Folder: / (root)

4. Click "Save"

5. Wait 1-2 minutes

6. Your portal will be live at:
   https://YOUR_USERNAME.github.io/error-categorization-portal/
